# Матрица ролей и разрешений (предложение)

> Статус: **проект, не реализовано в коде.** Роли уже есть в модели
> `tenants.Membership` (`owner` / `manager` / `cashier`), но проверки прав
> в API пока не подключены. Ниже — предлагаемая матрица для следующего шага.

## Роли

| Роль | Кто это | Основной интерфейс |
|------|---------|--------------------|
| **owner** | Владелец бизнеса (тенанта) | Админка + все разделы |
| **manager** | Товаровед / администратор | Каталог, склад, приёмка, отчёты |
| **cashier** | Продавец на кассе | Только касса (POS) |

## Матрица (ресурс × действие → роль)

Легенда: ✅ разрешено · ⬜ запрещено

| Ресурс | Действие | owner | manager | cashier |
|--------|----------|:-----:|:-------:|:-------:|
| Каталог (товары/варианты) | чтение | ✅ | ✅ | ✅ |
| Каталог | создание/изменение | ✅ | ✅ | ⬜ |
| Каталог | удаление | ✅ | ✅ | ⬜ |
| Голосовой ввод / quick-product | создать товар | ✅ | ✅ | ⬜ |
| Склад / остатки | чтение | ✅ | ✅ | ✅ |
| Приёмка (Receipt) | провести | ✅ | ✅ | ⬜ |
| Приёмка | история/чтение | ✅ | ✅ | ⬜ |
| Касса — смена | открыть/закрыть | ✅ | ✅ | ✅ |
| Касса — чек | провести продажу | ✅ | ✅ | ✅ |
| Касса — Z-отчёт | смотреть свой | ✅ | ✅ | ✅ |
| Возвраты (будущее) | оформить | ✅ | ✅ | ✅* |
| Отчёты (продажи/прибыль) | смотреть | ✅ | ✅ | ⬜ |
| Справочники (склады, кассы, филиалы) | изменять | ✅ | ✅ | ⬜ |
| Пользователи / членства | приглашать, менять роли | ✅ | ⬜ | ⬜ |
| Настройки бизнеса (тенанта) | менять | ✅ | ⬜ | ⬜ |

`*` возвраты кассиром — предлагается разрешить только в рамках своей открытой
смены; крупные возвраты/без чека — по роли manager/owner (уточнить позже).

## Принципы реализации (когда будем делать)

1. **Источник истины — `Membership.role` текущего тенанта** (уже в `request.membership`).
2. DRF-permission класс `HasTenantRole(*roles)` + пер-вьюсетные требования; по
   умолчанию — чтение всем ролям тенанта, запись — по матрице.
3. **cashier не имеет доступа** к разделам «Товары/Приёмка/Отчёты» — скрывать в
   меню фронта по `role` из `/api/auth/me/` и дублировать проверкой на бэке.
4. Действия, меняющие деньги/остатки, логировать с `actor` (кто провёл).
5. `owner` — единственный, кто управляет пользователями и настройками тенанта.
